# Plugging in real data

Every app in this repo runs end-to-end on synthetic-but-plausible data generated by its own `data/generate.py`. The compute pipeline (LLM calls, processing, UI, tool definitions) is real; the data is the swap point.

This doc tells you what to swap per app to point at real data — your hackathon dataset, your production data, anything.

## The three buckets

| Bucket | Effort | Apps |
|---|---|---|
| **A. Drop-in if columns match** | ~30 min: edit loader header, point at real CSV | VANGUARD, RIPTIDE, EMBER, MERIDIAN, CORSAIR, MARLIN, WILDFIRE |
| **B. Drop files in `data/`** | ~10 min: copy files, restart | SENTINEL, OPTIK, RAPTOR, FORGE, STRIDER |
| **C. Needs a real loader written** | Hours: real schema mapping + custom join logic | ANCHOR, WEATHERVANE |

## Bucket A — drop-in CSV (7 apps)

Each of these apps reads tabular data with a fixed column schema. To use real data, edit the constants at the top of `data/generate.py` (or replace it with a thin loader) so columns line up.

### VANGUARD — `09-vanguard-afcent-logistics/data/generate.py`
- **Real source:** AFCENT Logistics CSV (or any TMR transport dataset)
- **Required columns:** TMR ID, origin, destination, cargo class, weight, mode, priority, requested ship date
- **Swap:** Replace the `_generate_*` helpers with a `pd.read_csv(real_path)` call. Field names must match what `src/tools.py` expects.

### RIPTIDE — `04-riptide-fima-nfip-flood-claims/data/generate.py`
- **Real source:** FEMA OpenFEMA NFIP redacted claims
- **Required columns:** policy ID, claim date, claim amount, base name, base FIPS, peril, year built
- **Swap:** Drop FIMA's CSV in `data/raw/` and update the loader to read it.

### EMBER — `14-ember-firms-ukraine/data/generate.py`
- **Real source:** NASA FIRMS Ukraine 2-year extract (or any FIRMS export)
- **Required columns:** latitude, longitude, brightness, scan, track, acq_date, acq_time, satellite, confidence, frp
- **Swap:** Drop the FIRMS CSV (already in canonical schema) and the generator's column names line up automatically.

### MERIDIAN — `05-meridian-fema-supply-chain-resilience/data/generate.py`
- **Real source:** FEMA Supply Chain Resilience indicator dataset
- **Required columns:** node ID, lat, lon, type (port/depot/airfield), criticality score, climate exposure score, primary peril
- **Swap:** Replace generator with a loader reading FEMA's GIS export. The OPORD-format briefer downstream is schema-agnostic.

### CORSAIR — `06-corsair-imb-pirate-attacks/data/generate.py`
- **Real source:** IMB / NGA Pirate Attacks 1993–2020 dataset
- **Required columns:** date, lat, lon, attack type, vessel type, status, narrative
- **Swap:** The KDE forecaster + intel narrator work on the canonical IMB schema as-is. Drop the CSV, update the path constant.

### MARLIN — `01-marlin-ais-vessel-tracking/backend/`
- **Real source:** Spire / MarineCadastre AIS extract
- **Required columns:** MMSI, lat, lon, timestamp, sog, cog, vessel type, length, flag
- **Swap:** Edit the loader function in `backend/app.py` (`load_ais()` or similar) to read your real AIS export. Anomaly detection and SSE streaming are downstream of the loader.

### WILDFIRE — `13-wildfire-nasa-firms/`
- **Real source:** NASA FIRMS active-fire CSV
- **Required columns:** latitude, longitude, brightness, acq_date, acq_time, frp, confidence, satellite
- **Swap:** Same FIRMS schema as EMBER. Drop the CSV, update the path constant.

## Bucket B — drop files in a folder (5 apps)

These apps work on any matching file format. Just point at a folder.

### SENTINEL — `10-sentinel-military-object-detection/data/`
- **Real source:** Any military object detection imagery (.jpg / .png)
- **Swap:** Drop images in `data/imagery/`. Vision-language PID + SHA-256 hash-chained audit log work on any image input.

### OPTIK — `03-optik-coco8-vision-tm/data/`
- **Real source:** Any maintainer field photos (.jpg / .png)
- **Swap:** Drop photos in `data/samples/`. Replace `data/tm_snippets.json` with real TM excerpts when you have them.

### RAPTOR — `08-raptor-hit-uav-thermal/data/`
- **Real source:** Any thermal IR frame sequence (HIT-UAV format or any TIFF/PNG)
- **Swap:** Drop frames in `data/frames/`. Multi-frame vision-language reasoning works on any thermal/IR image format.

### FORGE — `02-forge-cwru-bearing-fault/data/`
- **Real source:** Any 12 kHz drive-end accelerometer CSV (CWRU schema or near-CWRU: 1-column time series sampled at 12 kHz)
- **Swap:** Drop CSVs in `data/signals/` named `<asset_id>.csv` (e.g. `MTVR-2491.csv`). Bearing-fault classifier + envelope spectrum + RUL estimator run on any 12 kHz vibration signal.

### STRIDER — `07-strider-goose-offroad-terrain/data/`
- **Real source:** Any off-road terrain frames (.jpg / .png)
- **Swap:** Drop frames in `data/terrain/`. Vision call returns structured terrain JSON regardless of source.

## Bucket C — needs a real loader (2 apps)

These make synthetic-specific assumptions that won't survive a column rename.

### ANCHOR — `11-anchor-msi-world-port-index/`
- **Real source:** MSI WPI port-capability shapefiles + custom port lat/lon join
- **Swap effort:** Write a loader that reads MSI's shapefile format (geopandas), extracts the port-capability fields ANCHOR's RAG corpus expects (berth depth, crane count, fuel availability, MPF compatibility), and emits a corpus JSON. The agentic RAG layer in `src/rag.py` and `src/app.py` is schema-agnostic but expects specific fields.

### WEATHERVANE — `12-weathervane-nasa-earthdata/`
- **Real source:** NASA Earthdata multi-band raster (HDF / NetCDF)
- **Swap effort:** Write a band-extraction loader (xarray + rioxarray) that reads NASA's raster formats and emits the time-series + grid the environmental-window agent expects. Today's generator emits flat JSON; real Earthdata is multi-dimensional.

## Verifying your swap

After replacing any app's data:

```bash
cd <app-folder>
python data/generate.py    # should run cleanly against real data path
streamlit run src/app.py   # or whichever launch command the app's README says
# Visually confirm: real records appear, real LLM call returns sensible output
```

If a downstream layer breaks (column missing, type mismatch), the error surfaces in the Streamlit traceback — fix at the loader, never deeper. The compute pipeline below the loader is data-agnostic by design.
