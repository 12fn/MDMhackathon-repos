# OPTIK ‚Äî On-Prem Tactical Image Knowledge

> Built on **[GAI](https://www.governmentacquisitions.com/) (Government Acquisitions, Inc.)** + **[Kamiwaza](https://www.kamiwaza.ai/)**. MIT-licensed; part of the [MDM 2026 Hackathon Templates](https://github.com/12fn/MDMhackathon-repos). See [`../ATTRIBUTION.md`](../ATTRIBUTION.md).

> Codename **OPTIK**. Agent #03 of 14. Hackathon: USMC LOGCOM CDAO AI Forum @ MDM 2026.

**One-liner:** turn any Marine's phone into an embedded technical librarian.

A Marine drops a photo of a broken thing ‚Äî a leaking CTIS valve, an unfamiliar coupling on an MTVR ‚Äî and OPTIK runs a vision-language model to identify it, then a cosine-similarity RAG over thirty synthetic Technical-Manual snippets to retrieve the right TM section, NSN, torque spec, and procedure. Output is a maintainer-grade brief that sounds like a senior Sergeant on the radio.

## Mission frame

Quoting **MARADMIN 131/26**:

> "AI-enabled models to anticipate equipment failure and improve readiness."

OPTIK is the eyeball-to-NSN bridge. Identification is step zero of predictive maintenance. Today: a junior Marine pages through 3,400+ pages of TM, calls a senior NCO over, or guesses at part numbers. With OPTIK: drop the photo, get the action.

## Dataset

- **COCO8** (Ultralytics object-detection mini-set, 8 images) ‚Äî provenance: https://docs.ultralytics.com/datasets/detect/coco8/ ‚Äî pulled live by `data/generate.py` from `https://ultralytics.com/assets/coco8.zip`. Used as drag-drop sample images so the demo always has something object-rich to point at.
- **30 synthetic TM snippets** ‚Äî generated by `data/generate.py` (seeded `random.Random(1776)`). Realistic format (TM 9-2320-XXX, NSN format `NNNN-CC-NNN-NNNN`, ft-lb / N-m torque, MIL-PRF lubricants). **Fully synthetic** ‚Äî not CUI. In production, swap with the LOGCOM-curated TM corpus on the authorized platform.

## Run it

```bash
cd apps/03-optik
pip install -r requirements.txt

# 1. Generate data + sample images + embedding index (one-shot, ~10 s):
python data/generate.py --embed

# 2. Launch:
python src/app.py
# -> http://localhost:3003
```

Drag a sample image (or the **placeholder_valve.jpg**), add an optional Marine's note, click **Identify + Retrieve TM**.

### Environment

**Primary path ‚Äî Kamiwaza-deployed inference (on-prem, IL5/IL6 ready):**

```bash
export KAMIWAZA_BASE_URL=https://kamiwaza.local/api/v1
export KAMIWAZA_API_KEY=...
# OPTIK now talks to your Kamiwaza-served vision + embedding models
# via the OpenAI-compatible mesh. Image and text never leave the wire.
```

**Developer fallback** ‚Äî when `KAMIWAZA_BASE_URL` is unset, the shared client
falls back to a direct OpenAI call (handy for laptop dev). Reads from repo root `.env`:

```bash
OPENAI_API_KEY=sk-...
OPENAI_PRIMARY_MODEL=gpt-5.4-mini
OPENAI_FALLBACK_MODELS=gpt-5-mini,gpt-4o-mini,gpt-4o
```

Zero code change between modes ‚Äî the model id stays the same; Kamiwaza maps it server-side.

## Hero AI play

A single **multimodal call to a Kamiwaza-deployed vision model** with `response_format=json_object` returns a typed detection record (scene, primary_subject, detections-with-bboxes, search_query). We embed the query against a Kamiwaza-served embedding model, cosine-search the TM corpus, take top-3 hits, and have a Kamiwaza-deployed reasoning model narrate the maintainer brief. Three bleeding-edge moves stacked: vision-language, structured output, RAG ‚Äî all behind the Kamiwaza Inference Mesh.
